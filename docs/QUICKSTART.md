# AgentNebula Quickstart

## Overview

AgentNebula is a universal infinite-loop agent workflow tool. It drives Claude Code to work through a task list autonomously, one session at a time, with full state persistence and a real-time web dashboard.

**Core concept**: Every workflow lives in a `.agent-nebula/` directory that contains 3 key files:

```
.agent-nebula/
├── config.yaml       # Project config + workflow settings
├── spec.md           # What you want done (input to the Initializer Agent)
├── task_list.json    # Generated by Initializer, or hand-written
├── progress.md       # Free-form progress notes (auto-updated by agents)
├── session_history/  # Markdown summaries of each session
└── session_messages/ # Full JSONL recordings of agent conversations
```

## Quick Setup (3 steps)

### Step 1: Install

```bash
cd /path/to/AgentNebula
pip install -e .
```

### Step 2: Initialize a workflow

```bash
# Option A: Auto-init in your project directory
cd /path/to/your/project
python -m agent_nebula init --spec spec.md

# Option B: Init with separate workflow directory
python -m agent_nebula init \
  -w ~/workflows/my-project/ \
  --cwd /path/to/your/project \
  --spec spec.md \
  --name "My Project"
```

### Step 3: Run

```bash
python -m agent_nebula run -w .agent-nebula
# Dashboard opens at http://localhost:8765
```

## Two Ways to Create task_list.json

### Way 1: Let the Initializer Agent generate it

Write a `spec.md` describing what you want done. When you run `agent-nebula run`, the Initializer Agent (Phase 1) reads the spec and generates `task_list.json` automatically.

```bash
python -m agent_nebula init --spec my_spec.md
python -m agent_nebula run -w .agent-nebula
```

### Way 2: Hand-write task_list.json

For full control, create `task_list.json` manually. See `docs/templates/task_list_template.json` for the format.

```bash
# After creating .agent-nebula/config.yaml and task_list.json manually:
python -m agent_nebula run -w .agent-nebula
```

## CLI Reference

```bash
# Initialize
python -m agent_nebula init [-w DIR] [--cwd DIR] [--spec FILE] [--name NAME]

# Run the infinite loop
python -m agent_nebula run [-w DIR] [--spec FILE] [--max-sessions N] [--port PORT] [--no-dashboard]

# Check status
python -m agent_nebula status [-w DIR]
```

| Flag | Description |
|------|-------------|
| `-w, --workflow-dir` | Path to the `.agent-nebula/` directory |
| `--cwd` | Working directory for Claude agent (default: current dir) |
| `--spec` | Path to a spec file |
| `--max-sessions N` | Limit sessions (-1 = infinite) |
| `--port PORT` | Dashboard web port (default: 8765) |
| `--no-dashboard` | Disable the web dashboard |

## How It Works

```
┌────────────────────────────────┐
│      python -m agent_nebula run │
│                                │
│  1. Has task_list.json?        │
│     NO → Phase 1: Initializer  │
│           Read spec.md         │
│           Generate tasks       │
│     YES → Skip to Phase 2     │
│                                │
│  2. Phase 2: Worker Loop       │
│     while (pending tasks):     │
│       task = next_pending()    │
│       session = claude(task)   │
│       update task_list.json    │
│       save session JSONL       │
│       sleep(3s)                │
│                                │
│  Dashboard: localhost:8765     │
│  Ctrl+C: graceful stop        │
└────────────────────────────────┘
```

## Key Design: workflow_dir vs cwd

- **workflow_dir**: Where AgentNebula state lives (config, tasks, sessions)
- **cwd**: Where the Claude agent actually reads/writes project files

These are independent. A workflow in `~/workflows/docs/` can drive Claude to work in `/opt/game-engine/`.

Set `cwd` in `config.yaml`:
```yaml
project:
  cwd: /path/to/your/project
```

## Dashboard

The web dashboard is fully project-agnostic. It reads whatever `.agent-nebula/` directory you point it at:

- **Main view**: Task grid with status, progress bar, session history
- **Task detail**: Click any task card to see metadata, dependencies, notes
- **Session detail**: Click session link to view full agent conversation (JSONL)
- **Live mode**: Active sessions stream in real-time via WebSocket
